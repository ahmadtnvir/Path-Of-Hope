Important FAQ

What is Document Object Model (DOM)?

The Document Object Model (DOM) is a programming interface for web documents. It represents the structure of a webpage as a tree of objects (like elements, attributes, and text), allowing developers to access, modify, add, or delete elements dynamically using JavaScript. Essentially, the DOM connects web content (HTML/CSS) with scripts for interactivity.

How do you select an element from the DOM?

You can select an element from the DOM using methods like: 1. document.getElementById("id")` for selecting by ID. 2.document.querySelector("selector")` for selecting the first element by a CSS selector.

What is event delegation in the context of the DOM and why is it useful?

Event delegation is a technique in the DOM where you attach a single event listener to a parent element instead of multiple listeners on child elements. The event listener can capture events from any of its child elements because events bubble up through the DOM tree. This is useful because it improves performance by reducing the number of event listeners and allows dynamically added elements to be handled without needing to attach new listeners.

How do you manipulate an element's attributes and styles using the DOM?

You can manipulate an element's attributes using methods like `setAttribute()` and `getAttribute()` or directly by accessing the properties (e.g., `element.id = "newId"`). To manipulate styles, you can use the `style` property (e.g., `element.style.color = "blue"`) or add/remove CSS classes using `classList` (e.g., `element.classList.add("newClass")`).
